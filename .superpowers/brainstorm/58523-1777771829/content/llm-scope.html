LLM 配置 + 聚类/情感：实现范围选择

点击选择一个交付范围。默认推荐先做“全局 LLM + 单次运行报告强化”，再迭代跨视频聚类。

A

全局 LLM 配置 + 单次运行展示（推荐）

Settings 配 model/baseUrl/key（加密存储），运行时自动注入；报告页展示 mvp_analysis.json（共识/争议、知识点、建议）。

Pros

  • 改动最小，风险低
  • Key 不落盘明文（env 注入）

Cons

  • 不包含跨视频聚类/导出
B

加上“跨视频知识库聚合”（规则版）

在 Reports 页提供多选 run，一键生成 kb_summary.md（规则去重+标签统计+社区摘要），并支持导出。

Pros

  • 快速满足“一键导出”
  • 不依赖 LLM，成本可控

Cons

  • 不支持用户自定义分类指令
C

完整：跨视频聚类/情感 + 自定义分类指令 + 导出

增加一个“聚合/分类”入口：用户输入指令（例如“按营销方法/受众/情绪分类”），LLM 输出分组、情感与可执行建议，并导出为 Markdown/JSON。

Pros

  • 最贴合你的最终目标
  • 分类规则由用户驱动，灵活

Cons

  • 范围更大：需要新 pipeline + UI + 存储/安全设计
  • 需要明确“输入数据范围/输出格式”
