# Phase 3.5（OCR 画面文字提取）Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为 dy/mvp 流程新增 OCR 画面文字提取能力（cv2 抽帧 + PaddleOCR 可选依赖），实现 ASR+评论+画面文字三重输入的 LLM 分析，并产出 ocr_summary、报告新增章节、以及 OCR 缓存复用/降级。

**Architecture:** 新增 `OCRService` 负责抽帧与 OCR 推理；`OCRPostprocessor` 做去重/排序/截断与摘要；在 `MVPPipeline` 与 `SingleVideoRunner` 中实现“下载后 ASR+OCR 并行”，并将 `ocr_text/ocr_summary` 写入 mvp_output；`AnalysisPipeline` 从 mvp_output 透传 ocr_text；LLMAnalyzer 的 knowledge_extract 融合 ocr_text 与 valuable_comments；最终增强 report_renderer 与 KnowledgeBase 输出。

**Tech Stack:** Python, asyncio, OpenCV (cv2), PaddleOCR (optional), pytest
