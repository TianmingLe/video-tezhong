# OmniScraper Desktop 用户手册（Beta）

## 安装与更新

### 安装

- 下载并运行对应系统的安装包（Windows/macOS/Linux）。
- 首次启动会进入引导流程（Onboarding），用于检查本机 Python 环境并完成必要的初始配置。

### 自动更新

- 应用支持自动更新下载。
- 你也可以在「设置」页点击「检查更新」进行手动检查。

## 核心功能

### 任务配置与运行

- 在「任务」页选择脚本与场景，填写环境变量（如需要）。
- 提交后任务进入队列执行，支持并发与排队。
- 支持取消正在运行的任务。

### 队列管理

- 应用会展示当前队列状态（运行中与排队数量）。
- 任务结束后会生成运行记录，可在「报告」页查看历史。

### 日志查看与导出

- 任务运行时日志实时输出，并提供搜索/按级别筛选。
- 报告页支持按 chunk 读取归档日志并展示关键事件摘要。
- 可在报告页点击「导出完整日志 (.log)」导出到本地文件。

### 托盘操作

- 应用支持托盘驻留与常用操作（显示/隐藏窗口、任务相关操作）。
- 可在「设置」页配置托盘左键行为。

## 常见问题（FAQ）

### 1) Python 环境检测失败

- macOS/Linux：请确认 `python3` 可在终端直接运行 `python3 --version`。
- Windows：请确认 `python --version` 可用，并且已将 Python 加入 PATH。
- 如你有多个 Python 版本，建议优先使用系统可直接调用的版本。

### 2) 如何导出日志？

- 进入「报告」→ 选择对应 RunID → 点击「导出完整日志 (.log)」。
- 若归档日志文件不存在，应用会使用运行期内存日志作为 fallback 导出。

### 3) 任务卡顿/无响应怎么办？

- 先尝试「取消」任务，再重新提交。
- 如果多次复现，请通过「设置」→「反馈问题」生成反馈文本并提交 Issue（见下文）。

## 反馈指南（Beta）

### 提交 Issue（推荐）

1. 打开应用 → 进入「设置」→ 点击「反馈问题」
2. 填写“问题描述”（可选，但建议包含复现步骤、期望结果与实际结果）
3. 点击「生成并复制」
4. 打开 GitHub 新建 Issue 页面并粘贴：
   - https://github.com/TianmingLe/video-tezhong/issues/new

复制的内容会包含：
- System Info（platform/arch/appVersion/nodeVersion/electronVersion）
- Crash Reports 摘要（若存在）
- Last Task（最近一次任务运行上下文）
- User Input（你的问题描述）
